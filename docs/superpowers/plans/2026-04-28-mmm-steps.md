# MMM-Steps Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-28-mmm-steps-design.md`
**Goal:** Show today's iPhone step count on the mirror, sourced from a Home Assistant pedometer sensor populated by the iOS Companion app, with a configurable daily goal and a stale-data hint.

**Order of work:** iPhone → HA wiring → module skeleton on Pi → fetch logic → rendering → wire into config.js + pages.js → mirror files back into this repo.

**All Pi commands run via SSH:** `ssh admin@10.0.0.249`

### Task 1: Enable iPhone Pedometer sensor and verify HA entity

**Files:** none

- [ ] **Step 1:** On the iPhone, open the Home Assistant Companion app:
      *App Configuration → Sensors → Pedometer → Enable*. Confirm
      Background App Refresh and Location: Always are granted (Settings
      → Home Assistant → Background App Refresh / Location).

- [ ] **Step 2:** In HA *Developer Tools → States*, search for `steps`
      and confirm the entity exists. Typical IDs:

      - `sensor.<device_name>_steps` (e.g. `sensor.iphone_domes_steps`)

      Note the exact entity ID — it goes into `stepsEntity` in config.

- [ ] **Step 3:** Smoke-test the HA REST endpoint from the Pi reusing
      the existing reminders token:

```bash
ssh admin@10.0.0.249
TOKEN="$(grep -oE 'haToken: "[^"]+"' ~/MagicMirror/config/config.js | head -1 | cut -d'"' -f2)"
ENTITY="sensor.iphone_domes_steps"   # adjust to your entity
curl -sS -w "\nHTTP %{http_code}\n" \
  -H "Authorization: Bearer $TOKEN" \
  "http://homeassistant.local:8123/api/states/$ENTITY"
```

Expected: HTTP 200, JSON with a numeric `state` and `last_updated`.
If `state` is `"unknown"` / `"unavailable"`, walk a few steps with the
phone unlocked, then retry — the sensor only reports once Apple Health
has a value for the day.

### Task 3: Implement fetch logic

**Files:**
- Modify: `~/MagicMirror/modules/MMM-Steps/node_helper.js`

- [ ] **Step 1:** In `_tick`, build the URL:

```js
const url = this.config.haUrl.replace(/\/$/, "")
          + "/api/states/" + encodeURIComponent(this.config.stepsEntity);
```

- [ ] **Step 2:** Fetch with `Authorization: Bearer ${haToken}`. Treat:

  - HTTP 200 → parse JSON, continue.
  - HTTP 401 → throw `"HA token expired"`.
  - HTTP 404 → throw `"entity not found: <stepsEntity>"`.
  - Other non-2xx → throw `HA HTTP <status> <first 120 chars>`.
  - Network error → propagate `err.message`.

- [ ] **Step 3:** Parse the state:

```js
const raw = (j.state || "").trim();
const isMissing = raw === "" || raw === "unknown" || raw === "unavailable";
const steps = isMissing ? null : Number(raw);
if (steps !== null && !Number.isFinite(steps)) {
  throw new Error("non-numeric state: " + raw);
}
const lastUpdated = j.last_updated || j.last_changed || null;
```

- [ ] **Step 4:** Emit:

```js
this.sendSocketNotification("MMST_DATA", {
  steps,           // number or null
  lastUpdated,     // ISO string or null
  generatedAt: new Date().toISOString()
});
```

On error: `sendSocketNotification("MMST_ERROR", String(err.message || err))`.

- [ ] **Step 5:** Add a `MMST_REFRESH` handler that calls `_tick()`
      immediately (used by the frontend's midnight rollover).

- [ ] **Step 6:** Verify with a one-shot dry run on the Pi (without MM):

```bash
node -e '
const TOKEN = "PASTE_HA_TOKEN_HERE";
const ENTITY = "sensor.iphone_domes_steps";
fetch("http://homeassistant.local:8123/api/states/" + encodeURIComponent(ENTITY),
  { headers: { Authorization: "Bearer " + TOKEN } })
  .then(r => r.json())
  .then(j => console.log("steps=", j.state, "updated=", j.last_updated));
'
```

Expected: prints today's step count and a recent ISO timestamp.

### Task 4: Implement frontend rendering

**Files:**
- Modify: `~/MagicMirror/modules/MMM-Steps/MMM-Steps.js`
- Modify: `~/MagicMirror/modules/MMM-Steps/MMM-Steps.css`

- [ ] **Step 1:** State machine in `getDom()`:

  - `this.errorMsg` set → render `.mmst-status` with the error;
    if `this.lastSteps != null`, also render the muted last-good count
    underneath.
  - `this.data === null && this.lastSteps == null` → render
    `.mmst-status` "čekám na data" / "waiting for data".
  - Otherwise → render `.mmst-count`, optional `.mmst-goal` block (only
    if `goal > 0`), and `.mmst-time`. Keep `lastSteps` as a fallback if
    the current `data.steps` is `null` (sensor briefly unavailable).

- [ ] **Step 2:** Number formatting:

```js
const fmt = new Intl.NumberFormat(
  this.config.language === "cs" ? "cs-CZ" : "en-GB"
);
countEl.textContent = fmt.format(steps);
```

- [ ] **Step 3:** Goal block — fill ratio capped at 1, label uncapped:

```js
const ratio = Math.max(0, Math.min(1, steps / this.config.goal));
bar.style.background =
  `linear-gradient(to right, #fff 0%, #fff ${ratio*100}%, #333 ${ratio*100}%, #333 100%)`;
const pct = Math.round((steps / this.config.goal) * 100);
label.textContent = this._lang("goal", { pct, goal: fmt.format(this.config.goal) });
```

- [ ] **Step 4:** Stale detection:

```js
const ageMin = (Date.now() - Date.parse(this.data.lastUpdated)) / 60000;
const isStale = ageMin > this.config.staleAfterMin;
if (isStale) wrap.classList.add("mmst-stale");
timeEl.textContent = this._lang(isStale ? "stale" : "updated",
  { hhmm: this._fmtTime(this.data.lastUpdated) });
```

- [ ] **Step 5:** Local-midnight rollover — same pattern as MMM-Spending:

```js
this.dayTick = setInterval(() => {
  const today = this._localDayKey();
  if (today !== this._currentDay) {
    this._currentDay = today;
    this.sendSocketNotification("MMST_REFRESH");
  }
}, 60_000);
```

- [ ] **Step 6:** Lang dictionary covers: `waiting`, `updated`, `stale`,
      `goal` (with `{pct}` / `{goal}` placeholders). cs and en.

- [ ] **Step 7:** Visual smoke test on the Pi:

```bash
pm2 restart MagicMirror
pm2 logs MagicMirror --lines 30
```

Open the mirror screen — module should appear with today's count, the
goal bar and the "aktualizováno HH:MM" line.

### Task 6: README

**Files:**
- Create: `~/MagicMirror/modules/MMM-Steps/README.md`

- [ ] **Step 1:** Document, in this order:

  1. What it does (one paragraph).
  2. **iPhone setup** — install HA Companion app, log in, *App
     Configuration → Sensors → Pedometer → Enable*. Grant Background
     App Refresh and Location: Always.
  3. **Find your entity ID** — HA → *Developer Tools → States* → search
     `steps`. Copy the exact `sensor.<device>_steps` ID.
  4. **HA token** — reuse the long-lived token from MMM-HA-Reminders;
     if you don't have one yet, HA → profile → Long-Lived Access
     Tokens → Create Token.
  5. **Notes** — first reading can be `unknown` until the phone walks
     a few steps after enabling. Background App Refresh must be on or
     the value freezes when the screen is off. No npm deps. Native
     fetch (Node 18+).
