# MMM-Spending Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Spec:** `docs/superpowers/specs/2026-04-27-spending-design.md`
**Goal:** Show today's spending on the mirror, sourced from Wallet by BudgetBakers' REST API, summed across personal accounts (Běžný + Hotovost), with transfers excluded and investments included.

**Order of work:** Module skeleton on Pi → fetch + filter logic → rendering → wire into config.js + pages.js → mirror files back into this repo.

### Task 1: Generate API token and confirm endpoint

**Files:** none

- [ ] **Step 1:** In a browser, log into `web.budgetbakers.com`,
      Settings → API Tokens → create a new token named
      `magicmirror-pi`. Copy it once — UI will not show it again.

- [ ] **Step 2:** Smoke-test the token from the Pi:

```bash
ssh admin@10.0.0.249
TOKEN="<paste token>"
curl -sS -w "\nHTTP %{http_code}\n" \
  -H "Authorization: Bearer $TOKEN" \
  "https://rest.budgetbakers.com/wallet/v1/api/records?recordDate=gte.$(date -u +%Y-%m-%d)T00:00:00Z&limit=10"
```

Expected: HTTP 200 and a JSON body with `records: [...]`.
If 401 — token is wrong. If `{"error": "unknown query parameter ..."}` — re-check param name (`recordDate=gte.<ISO>` is the verified form).

- [ ] **Step 3:** Save the token to a file readable only by `admin`,
      to be sourced from config.js later:

```bash
mkdir -p ~/.secrets
chmod 700 ~/.secrets
echo "$TOKEN" > ~/.secrets/budgetbakers_token
chmod 600 ~/.secrets/budgetbakers_token
```

(We don't `export` it from a shell rc — config.js reads the file
synchronously at MM startup.)

### Task 3: Implement fetch + filter + sum

**Files:**
- Modify: `~/MagicMirror/modules/MMM-Spending/node_helper.js`

- [ ] **Step 1:** In `_tick`, build the URL:

  - `start = new Date(today 00:00 in config.timezone).toISOString()`
    — convert local-midnight to UTC ISO. Use
    `Intl.DateTimeFormat("en-CA", { timeZone, ... })` to get the
    YYYY-MM-DD then construct a Date.
  - `url = ${apiBase}/records?recordDate=gte.${encodeURIComponent(start)}&limit=200`

- [ ] **Step 2:** Fetch with `Authorization: Bearer ${apiToken}`. On
      non-2xx, throw with status + first 120 chars of body.

- [ ] **Step 3:** Apply filters in this order:

  1. `r.recordType === "expense"`
  2. `includeAccountIds.includes(r.accountId)`
  3. `!excludeEnvelopeIds.includes(r.category && r.category.envelopeId)`
  4. `Date.parse(r.recordDate) >= startMillis` (re-check client-side
     so 00:00–02:00 local-time records aren't dropped for the previous
     UTC day)

- [ ] **Step 4:** Compute aggregate:

```js
const filtered = records.filter(...);
const total = filtered.reduce((s, r) => s + Math.abs(r.amount.value), 0);
const recent = filtered
  .slice()
  .sort((a, b) => Date.parse(b.recordDate) - Date.parse(a.recordDate))
  .slice(0, this.config.recentCount)
  .map(r => ({
    payee: r.payee || r.payer || "—",
    amount: Math.abs(r.amount.value),
    currency: r.amount.currencyCode
  }));
```

- [ ] **Step 5:** `sendSocketNotification("MMSP_DATA", { total, currency: this.config.currency, count: filtered.length, recent, generatedAt: new Date().toISOString() })`.
      On error, `sendSocketNotification("MMSP_ERROR", String(err.message || err))`.

- [ ] **Step 6:** Verify with a one-shot dry run on the Pi (without MM):

```bash
node -e '
const TOKEN = require("fs").readFileSync(require("os").homedir()+"/.secrets/budgetbakers_token","utf8").trim();
const start = new Date(); start.setHours(0,0,0,0);
const url = `https://rest.budgetbakers.com/wallet/v1/api/records?recordDate=gte.${encodeURIComponent(start.toISOString())}&limit=200`;
fetch(url, { headers: { Authorization: "Bearer " + TOKEN } })
  .then(r => r.json())
  .then(j => {
    const inc = ["ae939246-f1d5-4f14-bce1-c04357e0e77d","c989cb70-8e88-4fd5-a103-b3f45278dc41"];
    const f = j.records.filter(r => r.recordType==="expense" && inc.includes(r.accountId) && (r.category||{}).envelopeId !== 20001);
    console.log("count=", f.length, "total=", f.reduce((s,r)=>s+Math.abs(r.amount.value),0));
    f.forEach(r => console.log(" ", r.recordDate, r.payee||r.payer, r.amount.value));
  });
'
```

Expected: a count and total that matches what the user sees in the
Wallet app for today (excluding business account and transfers).

**Files:**
- Modify: `~/MagicMirror/modules/MMM-Spending/MMM-Spending.js`
- Modify: `~/MagicMirror/modules/MMM-Spending/MMM-Spending.css`

- [ ] **Step 1:** In `getDom()`:

  - If `this.errorMsg` is set → render `.mmsp-status` row with the
    error and (if available) a small "naposledy HH:MM" line.
  - If `this.data === null` → render `.mmsp-status` "načítání…".
  - Else render: `.mmsp-total`, `.mmsp-count`, then up to
    `recentCount` `.mmsp-recent` rows.

- [ ] **Step 2:** Number formatting via
      `new Intl.NumberFormat(this.config.language === "cs" ? "cs-CZ" : "en-GB", { style: "currency", currency: this.data.currency, maximumFractionDigits: 2 })`.
      Recent items show the same format but with
      `maximumFractionDigits: 0`.

- [ ] **Step 3:** Czech-grammar count helper:

```js
_pluralCs(n) {
  if (n === 1) return "transakce";
  if (n >= 2 && n <= 4) return "transakce";
  return "transakcí";
}
```

Render `${count} ${_pluralCs(count)} dnes`. English path: `${count} transaction${count===1?"":"s"} today`.

- [ ] **Step 4:** Local-midnight rollover:

```js
this.dayTick = setInterval(() => {
  const today = this._localDayKey();
  if (today !== this._currentDay) {
    this._currentDay = today;
    this.sendSocketNotification("MMSP_REFRESH");  // node_helper triggers _tick on this
  }
}, 60_000);
```

(Add `MMSP_REFRESH` handling in `node_helper.js` to call `_tick()` immediately.)
