## Goal

Add face recognition to MagicMirror on Raspberry Pi so that when Domes stands in front of the mirror, it switches to his profile (shows greeting + personal widgets). After 2 minutes without a detected face, it returns to the anonymous profile.

## Architecture

Three components working together:

1. **MMM-Face-Reco-DNN** — reads the RPi Camera Module, runs face recognition using OpenCV DNN, emits MagicMirror notifications
2. **MMM-Greeting** — new lightweight module (~50 lines JS) that listens for notifications and displays "Hi, Domes" at `top_center`
3. **Profile switching via `classes`** — existing modules use the `classes` config key to show/hide based on the recognized user

```
RPi Camera → MMM-Face-Reco-DNN → CURRENT_USER notification → MMM-Greeting (show "Hi, Domes")
                                                             → modules with classes: "Domes" (show)
                               → EVERYBODY_LEAVES notification → MMM-Greeting (hide)
                                                               → modules with classes: "Domes" (hide)
```

## Components

### MMM-Face-Reco-DNN

- Installed at `~/MagicMirror/modules/MMM-Face-Reco-DNN`
- Uses OpenCV DNN for face detection + recognition
- Config:
  - `interval: 2` — capture frame every 2 seconds
  - `usernameTimeout: 120000` — 2 minutes before switching to anonymous profile
  - `users: ["Domes"]` — single profile
- Training data: `dataset/Domes/` — ~10 photos of Domes from various angles/lighting
- Encoded faces: `encoded_faces.pickle` — generated by `python3 tools/encode_faces.py`

### MMM-Greeting

- New module created at `~/MagicMirror/modules/MMM-Greeting`
- Position: `top_center`
- Hidden by default
- On `CURRENT_USER` notification with `username: "Domes"` → shows "Hi, Domes"
- On `EVERYBODY_LEAVES` notification → hides itself
- Simple DOM update, no external dependencies

### Profile switching in config.js

- Modules visible to everyone (no `classes`): `clock`, `weather`, `newsfeed`, `updatenotification`
- Modules visible only to Domes (`classes: "Domes"`): `MMM-GoogleCalendar`
- MMM-Greeting always rendered but hidden/shown via notifications

## Data Flow

**Recognition flow:**
1. Camera captures frame every 2s
2. MMM-Face-Reco-DNN compares frame against `dataset/Domes/1.jpg ... N.jpg`
3. Match found → emits `CURRENT_USER: { username: "Domes" }`
4. MMM-Greeting renders "Hi, Domes" at top center
5. Modules with `classes: "Domes"` become visible (Google Calendar etc.)

**Timeout / leave flow:**
1. No face detected for 2 minutes
2. MMM-Face-Reco-DNN emits `EVERYBODY_LEAVES`
3. MMM-Greeting hides
4. Modules with `classes: "Domes"` hide, only public widgets remain

## Training Process

1. Take ~10 photos of Domes (different angles, lighting conditions)
2. Save to `modules/MMM-Face-Reco-DNN/dataset/Domes/1.jpg` ... `10.jpg`
3. Run `python3 tools/encode_faces.py` to generate `encoded_faces.pickle`
4. Restart MagicMirror

## Dependencies

- Python 3 + OpenCV (`python3-opencv`) on Raspberry Pi
- `face_recognition` Python package
- Node.js (already present for MagicMirror)
- RPi Camera Module (ribbon cable, already connected)

## Out of Scope

- Multiple users
- Remote profile management UI
- Face enrollment via MagicMirror itself
